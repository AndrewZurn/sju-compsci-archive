Lab 4: Simple numeric and character datatypes

The program you will use for this lab shows the internal representation of data on a small computer. Launch the data rep software by going to f-Applications -->CSBSJU -->Computer Science --> Launch Page Apps --> Data Representation
 
This program is a tool to assist you in learning several different methods that are used to represent data inside of a computer. You change the values of the bit patterns by clicking on the individual bits. They toggle between zero and one when clicked and the corresponding value is displayed in the text boxes on the right side of the screen. You can also type numeric values directly into the white text boxes on the right of the screen and the value typed will be translated into the appropriate bit pattern when you press the “Enter” key.
 
The type of data being displayed and the length of the bit patterns can both be changed to suit the user’s needs. There is a button on the bottom, just right of the quit button that says 8 bit words. To change the word length, you click on it. It will change to 16, then to 32, then to 64, and, finally, back to 8. Cycle through these categories using the button. There is also a button labeled Hexadecimal that can be used to select the type of data to be displayed. This is the type button. Click on it to see what happens.

Exercise 4: Floating Point numbers

In this part of the lab we will work with a "phony" 8-bit floating point scheme we developed for teaching purposes. To get to it, click on DataType/Real. Those 8 bits have been split into three parts: a 1-bit spot to hold the sign, a 4-bit spot to hold the exponent, and a 3-bit spot to hold the "mantissa".


For example, suppose we tried the number -1.75. In binary, this would be 1|0111|110 or -1.11 (1+1/2+1/4) or -1.11 * 20. This means that you should see a 1 in the sign position and 110 (or .75) in the mantissa position. (Remember the 1 before the binary point is not shown - it is hidden). Since we ended up with -1.11 * 20 then the exponents is zero (b/c 2 is raised to the power 0), however, our software shows a binary value of 0111 for the exponent! Given this, can you guess the shift value (or the offset) for this new design (which used to be 4 in the old design that we learned in class)?
 
Enter 33 in the white box and hit Enter. What happened? Try 32. Try 34. What is going on? Think about the possible bit patterns.

Make sure 32 is in the white area. Now click double. Hit double again. Hit it again. What do you see? Now click on one of the mantissa bits. You should see a symbol NaN which stands for Not a Number. This is because there are no legal bit patterns where the exponent bits are 1111.


Enter the bit pattern 0 1000 111. It should have the value 3.75. Hit half. Hit it again. Keep hitting until the value doesn't change. This is the smallest value you can enter. Then enter 0 1110 111. This is the largest (finite) value you can enter.

Now open DrJava and open the file GreekCharacters.java that we have supplied for you. Run the program and look at the code to see how you could add Greek characters to your program if you wanted. You can do similar things with all the Unicode languages. Note that the char type is another primitive type that is used to represent a single character. The \u indicates that this is a Unicode character represented by the hexadecimal number that follows char. So the declaration of:
char c='\u0391' declares a variable of type char (character) with a Unicode value of 0391 ( binary value: 0000 0011 1001 0001).

A later statement of c='\u03B1' changes the value of the variable to be the Unicode value 03B1 (binary value (0000 0011 1011 00001).
