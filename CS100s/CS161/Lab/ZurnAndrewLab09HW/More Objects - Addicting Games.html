Open up a second browser window and go to the website http://www.mostaddicting.com/Addictive.html. Play the game several times until you have a good idea of how it works. Your goal is to build a game like this using ActiveObjects!

To build the game, you will need to create two Java files. The first file will be the file that will contain the class that runs the game itself (i.e. controller class). Let's call this file AddictingGame.java and the contained class AddictingGame (this class should extend FrameWindowController). The second file is for the ActiveObject class that will represent the 4 blue rectangles. Let's call the second file BlueBox.java and the class that it holds will be the class BlueBox which extends ActiveObject.

Step 1: In the AddictingGame class we will need the following:

·         Constants that represent the initial positions and sizes of the four blue rectangles (choose different locations and sizes as shown in the game on the website above) as well as the initial position and size of the red center rectangle,

·         Constants that represent the initial position and size of the white filled rectangle in the center of the canvas

·         Variable names for the four BlueBoxes as well as the red rectangle.

·         Variable names for two boolean variables to control the game: drag (initially false) and newGame (initially true)

·         Other constants and variables as needed

·         In begin we will need to:

o    create a black FilledRect to cover the entire canvas,

o    create the white FilledRect over the top of the black rectangle,

o    create a red FilledRect in the center of the canvas

·         In onMousePress we will need to check to see if the red rectangle contains the point and if it does:

o    set drag variable to true (tells us if we're pressing inside the red rectangle before dragging...otherwise, no dragging needs to take place and game shouldn’t start)

o    if this is a new game (we need to know if we’re pressing for the first time to start the game vs. pressing more than once during the same game)

§  set newGame variable to false (tells us if we're playing a new game or continuing an old game. In the latter case, pressing should not recreate the BlueBoxes again ... see next step below)

§  create the four blue BlueBoxes

o    We need to create the BlueBoxes in the onMousePress so that they don't start moving until we are ready. 

·         In onMouseDrag, as long as the red rectangle doesn't overlap the BlueBoxes (see GREEN highlighted note below) or go outside of the inner walls (i.e. walls of the inner white rectangle), we will need to drag the red rectangle around with the mouse. Otherwise, we reset the game to be ready to play again. The BlueBoxes should have ended their run method and been removed from the canvas and the red rectangle should be replaced in the center of the screen.

·         Also reset the game in onMouseExit

o    Instead of repeating your reset code twice, consider building a method to reset the game.

Step 2: In class BlueBox we will need the following:

o    You will need to define a constructor that will create a blue FilledRect for a BlueBox (allow for different sizes and locations for each one).

o    In the run method

§  You should start by pausing briefly so that the user can be ready when the BlueBoxes start to move.

§  You should then start your loop to move and pause the FilledRect (use constants for the moved distance and the pause).  Like the CrazyRectangles class from class on ActiveObjects (it is included in this lab folder), you will want to bounce off the outer walls of the canvas. Feel free to use any code from class CrazyRectangles. You will need to adjust the size and speed parameters of the BlueBoxes until you have a playable game. That is why we define them as constants so that they are easy to change

§  You will need to continue until the game is over

§  What class should decide when the game is over and how will you communicate between the game and each of the BlueBoxes?) --- Hint: consider adding a method stopMoving() in class BlueBox like we did in StoppingBall example from class (will need a boolean variable in the while loop of the run method in BlueBox which will be set to false by the controller when the game ends).

o    There is a method for filled rectangles defined as boolean overlaps(someRect) which checks whether parameter someRect overlaps a given rectangle. Consider defining a similar public boolean overlaps(FilledRect someRect) method in class which returns true if parameter someRect overlaps the BlueBox.
