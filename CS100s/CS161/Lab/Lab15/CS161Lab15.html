Part 1: Image manipulation

Images can be thought of as a 2-dimensional array of pixels, where each pixel has a color value. To manipulate an image is to manipulate its pixels. In a terminal window, go to the Lab15/Sample directory and start the program, ImageManipulator, by entering java ImageManipulator. Note that the combobox has multiple selections and as you change the selections, the image on the right changes. For some selections (the last two to be exact – Rectangular averaging and rotate) you must move the slider to see the effects.

It is your job to recreate those methods so that the application in Lab15 works just like the application in the Sample directory. To do so, you will need to do the following steps:

·        Create a 2-dimensional array to hold the color values. You may want to look at some of the other methods such as adjustSaturation to see how to declare and set your 2D array.

·        Copy the pixel color values from the original array (i.e., sourcePixels) to the new array but in the appropriate locations to flip the image horizontally or vertically

·        Call method displayPreview on the new array as done in other methods: displayPreview(transformed);

 

Note: Please note that image sizes are generally referred to as "width x height." That means that when we talk about the pixel grid, we will generally refer first to the number of columns and then to the number of rows. The sourcePixels 2-d array will be an array of [col][row], not [row][col] as we usually refer to arrays.

Part 2:  Command line parameters.

When we write a program for use from the command line, we need a "launching" class that has the method main in it. In general, the "launching" class looks as follows:

public class RunSomeOtherClass {
    public static void main(String [] args) {
       SomeOtherClass someObject=new SomeOtherClass(....);
       someObject.run();  //or, perhaps, some other methods in the class someObject might also be called.
    }
}

What we are going to discuss is the use of the String [] args inside the parentheses - the parameters for the main method. It turns out that the parameter args is what it says it is - that is, it is an array of strings. More importantly, it is an array of strings, each of which is something you typed on the command line after launching the program. For example, you would normally expect to type:

java RunSomeOtherClass

to cause this program to run. If you did, the args array would be an array with 0 elements in it. On the other hand, if you typed

java RunSomeOtherClass   firstthing    secondthing

the args array would be of length 2 and args[0] would be the string "firstthing" and args[1] would be the string "secondthing". You can then use refer to these strings inside of the main method.  What this means is that you can design your programs so that when the program is launched from the command line the users can specify things they want them to do.  Suppose we wanted to specify a file for our program to read data from. Then the user could specify the file name at the command line. The main method would then have the name of the file as args[0] and could use that information to oepn the file.
