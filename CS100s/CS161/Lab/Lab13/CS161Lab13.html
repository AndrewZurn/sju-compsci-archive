Lab 13: Recursive Structures

As last time, you should again program in pairs.  You should partner with the same person that you partnered with last time but now switch roles; whoever was not at the keyboard last time, should be at the keyboard this time. 

In this lab, you are to develop a recursive structure that would represent a target that has rings around a bullseye. The size of the target is defined by the user (by providing the radius of the largest target ring in a text field) and the number of rings around the target is determined by the size (the rings are a fixed distance apart). To see what the program would look like, change to the Sample subdirectory of your Lab13 directory, open a terminal shell, and enter the command: javac RunTarget.java (this creates the .class file which is the binary version of the code that the computer understands) and then the command: java RunTarget.  Try changing the radius in the bottom text field so that you can change the size of the target created when you click on the canvas. Your lab assignment is to develop the recursive structures to support the interface as demonstrated. Keep in mind that in this example, the center of the bullseye should be at the point where the canvas is clicked. You are given the file TargetController.java which tests your classes when done.

Interface file

In your Lab13 directory (not in the Sample subdirectory), create a file TargetInterface.java that specifies the interface TargetInterface. 

  •   In it you should include public methods move, moveTo, and contains.
  •   Also specify in this interface public constants for the smallest size of the ring (say 20) and call this constant MIN_OUTER, and for the gap distance between rings (say 10) and call this constant GAP_DISTANCE. Note that since the base class and the recursive class will eventually implement this interface, constants MIN_OUTER and GAP_DISTANCE are accessible in both classes provided they have public visibility in TargetInterface (BTW, in Java, everything included in an interface must be defined with public visibility). This is much better than having to define the constants twice (in base class as well as in the recursive class).

Base class File (BullsEye.java)

In this file, you should write the specification for the center BullsEye which implements the TargetInterface.  To do this you should:

  •    define any appropriate instance variables  (a filled oval to represent the center of bullseye)
  •   define a non-recursive constructor that creates the center of the bullseye (a filled oval)
  •    make your bullseye red
  •     implement the methods needed to support TargetInterface

Remember that the input number is the radius, not the width or height of the target!

Recursive class File (RingedTarget.java)

In this file, you should write the specification for the recursive class that implements TargetInterface. To do this, you should:

  •    define the appropriate instance variables
  •    create a recursive constructor that will create an outer ring  (a framed oval) and then the inner target (keep in mind that the point passed in as a parameter is the center of the target)
  •    implement the methods needed to support TargetInterface by making recursive calls
